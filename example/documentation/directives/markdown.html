Angular Markdown Directive Parse Markdown text as HTML

Directive Overview

This directive requires the use of the open source markdown parser for javascript marked you can get this from GitHub it should also have been downloaded to your bower repository, if you used bower to install the toolbelt. Simply add the library to your HTML template <script src="/path/to/marked.js"></script>, before initialising your application, if this library is not present a warning will show instead of the parsed markdown.

<div data-sys-markdown="[scope.variable]"></div>

data-sys-markdown="[ngModel]" Directive initialisation, note that you can pass a standard angular scope variables into the directive for it to monitor / parse.

Demo

HTML 

 <div class="row" data-ng-controller="markdownCtrl">
    <div class="col-sm-6">
        <strong>Enter Markdown</strong>
        <textarea class="form-control" rows="12" data-ng-model="markdown"></textarea>
    </div>
    <div class="col-sm-6">
        <strong>Dynamic Result (cool)</strong>
        <div data-sys-markdown="markdown"></div>
    </div>
</div>
JavaScript 

.controller('markdownCtrl', ['$scope', function($scope) {
    $scope.markdown = "### Hello World\n\n**Welcome to [markdown](https://daringfireball.net/projects/markdown/)**\n\nPlease edit the text in the input to try out the various markdown tags *live*";
}])

Tips

You can also just use an HTML tag to parse the markdown text, note however that 'whitespace' will effect output from the parser.


<sys-markdown>
### Markdown Text
Can auto link http://www.sysen.co.uk
</sys-markdown>


### Markdown Text
Can auto link http://www.sysen.co.uk

The directive also caters for options to be sent to the marked converter using the options attribute. Or you can do this via javascript sending new options to the markdownConverterProvider, during initialisation


<sys-markdown data-options="{ gfm: false }">
### Markdown Text
Can disable auto link (GitHub flavoured markup) http://www.sysen.co.uk
</sys-markdown>


### Markdown Text
Can disable auto link (GitHub flavoured markup) http://www.sysen.co.uk

Consider this: As this directive can work with scope variables, you could use it do allow inline editing of text and save updates to a backend (like a basic CMS)
